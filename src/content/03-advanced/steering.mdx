# Steering 配置指南

Steering 是 Claude Code 的一个高级功能，它允许你为团队定制 AI 的行为、建议和输出。通过 Steering，你可以确保 AI 遵循团队的编码规范、最佳实践和项目特定的要求。

## 什么是 Steering？

Steering 是一组配置规则，用于指导 Claude Code 的 AI 如何理解和响应你的请求。它包括：

- **编码规范**：命名约定、代码风格、项目结构
- **最佳实践**：设计模式、性能优化、安全性要求
- **项目特定规则**：技术栈、依赖项、特殊要求
- **团队指南**：沟通风格、文档标准、审查流程

## Steering 的优势

### 1. 一致的代码风格

确保 AI 生成的代码与团队的编码规范一致。

### 2. 更好的建议

AI 能够理解团队的特定需求和偏好，提供更相关的建议。

### 3. 知识共享

将团队的最佳实践和经验编码到 Steering 中，新成员可以快速学习。

### 4. 自动化合规性

确保所有代码都遵循团队的规范和最佳实践。

## 创建 Steering 文件

### 基本结构

Steering 文件位于 `.kiro/steering/` 目录中，使用 Markdown 格式：

```markdown
---
inclusion: always  # always, fileMatch, manual
fileMatchPattern: "src/**/*.ts"  # 可选，用于 fileMatch
---

# 团队编码规范

## 命名约定

- 变量和函数：camelCase
- 类和接口：PascalCase
- 常量：UPPER_SNAKE_CASE
- 文件名：kebab-case（除了组件）

## 代码风格

- 使用 2 个空格缩进
- 行长度限制：100 字符
- 使用分号
- 使用单引号

## 最佳实践

- 始终添加类型注解
- 编写单元测试
- 添加 JSDoc 注释
```

# 我们的编码规范

## 项目概述

这是一个 React + TypeScript 的前端项目。我们遵循以下规范来确保代码质量和一致性。

## 技术栈

- **框架**：React 18+
- **语言**：TypeScript 5+
- **样式**：Tailwind CSS
- **状态管理**：Zustand
- **测试**：Jest + React Testing Library

## 命名约定

### 文件和文件夹

- 组件文件：PascalCase（`UserProfile.tsx`）
- 工具函数：camelCase（`formatDate.ts`）
- 样式文件：kebab-case（`user-profile.css`）
- 测试文件：`*.test.ts` 或 `*.spec.ts`
- 文件夹：kebab-case（`user-profile/`）

### 代码

- 变量和函数：camelCase
- 类和接口：PascalCase
- 常量：UPPER_SNAKE_CASE
- 私有属性：前缀 `_`
- 布尔变量：前缀 `is`, `has`, `can`

### 示例

```typescript
// ✅ 好的命名
const MAX_RETRIES = 3;
const isLoading = false;
const getUserById = (id: string) => { ... };
interface UserProfile { ... }
class UserService { ... }

// ❌ 不好的命名
const max_retries = 3;
const loading = false;
const get_user_by_id = (id: string) => { ... };
interface userProfile { ... }
class userService { ... }
```

## 代码风格

### 缩进和格式

- 使用 2 个空格缩进（不使用 Tab）
- 行长度限制：100 字符
- 使用分号
- 使用单引号（字符串）
- 使用双引号（JSX 属性）

### 导入顺序

```typescript
// 1. 外部库
import React from 'react';
import { useEffect } from 'react';

// 2. 内部模块
import { UserService } from '@/services';
import { useUserStore } from '@/store';

// 3. 组件
import { UserCard } from '@/components';

// 4. 样式
import styles from './User.module.css';
```

### 类型注解

始终为函数参数和返回值添加类型注解：

```typescript
// ✅ 好的
function getUserById(id: string): Promise<User> {
  return fetch(`/api/users/${id}`).then(r => r.json());
}

// ❌ 不好的
function getUserById(id) {
  return fetch(`/api/users/${id}`).then(r => r.json());
}
```

## 性能指南

### 优化建议

- 使用 React.memo 避免不必要的重新渲染
- 使用 useMemo 和 useCallback 优化性能
- 代码分割和懒加载
- 图片优化和压缩
- 避免在渲染中创建新对象

### 性能指标

- 首屏加载时间：< 3 秒
- 交互时间：< 100ms
- 最大内容绘制：< 2.5 秒

## 安全性指南

### 常见安全问题

- 避免 XSS 攻击：不要使用 `dangerouslySetInnerHTML`
- 避免 CSRF 攻击：使用 CSRF 令牌
- 验证用户输入
- 不要在代码中硬编码敏感信息
- 使用 HTTPS

### 安全检查清单

- [ ] 所有用户输入都已验证
- [ ] 敏感数据已加密
- [ ] 没有硬编码的密钥或令牌
- [ ] 依赖项已更新到最新版本
- [ ] 已运行安全审计工具

## 常见问题

### Q: 如何在项目中应用 Steering？

**A:** 将 Steering 文件放在 `.kiro/steering/` 目录中，Claude Code 会自动应用这些规则。

### Q: 可以有多个 Steering 文件吗？

**A:** 可以。你可以为不同的文件类型或模块创建不同的 Steering 文件。

### Q: 如何更新 Steering 规则？

**A:** 编辑 `.kiro/steering/` 中的文件，Claude Code 会自动重新加载规则。

### Q: Steering 会强制执行吗？

**A:** Steering 是指导性的，不是强制性的。AI 会尽力遵循规则，但最终的决定权在你。
